The Campus Scheduling Challenge تحديات الحجز وجدولة الملاعب التقليدية

The Problem المشكلة

Traditional sports booking portals fail to manage court abuse, leaving physical facilities empty or monopolized. تفشل أنظمة حجز الملاعب التقليدية في التحكم بسلوك الاستخدام، مما يترك الملاعب فارغة أو محتكرة من فئة معينة.

  • Monopoly & Court Hogging: Specific groups booking courts back-to-back, locking others out. الاحتكار وحجز الملاعب المتكرر: قيام مجموعات معينة بحجز الفترات المميزة متتالية دون إتاحة الفرصة للآخرين.
  • No-Show Waste: Slots reserved but remaining empty when students fail to show up. هدر فترات الحجز (عدم الحضور): بقاء الملاعب فارغة رغم حجزها بسبب عدم حضور الطلاب، وحرمان الآخرين.
  • Check-In Fraud: Students checking in remotely (from dorms) instead of being at the facility. التسجيل الوهمي للحضور: تسجيل الحضور عن بُعد من الغرف السكنية دون الذهاب الفعلي للملعب.
  • Teammate Discovery Friction: Solo players struggling to find teammates or match student IDs. صعوبة إيجاد الفرق: عجز اللاعب الفردي عن تشكيل فريق أو جمع هويات الطلاب اللازمة لإتمام الحجز.

Monopoliesالاحتكار

Same groups booking prime slots consecutively. تكرار نفس الطلاب لحجز الساعات الذهبية باستمرار.

Remote Fraudالحضور الوهمي

Checking in on maps without walking to the court. تسجيل الحضور كذباً دون التواجد الجسدي بالملعب.

LFG Frictionعقبات تشكيل الفرق

Solo players locked out by player quota validation requirements. صعوبة حجز الملاعب لعدم وجود زملاء كافيين لتسجيلهم.

Language Barriersحواجز اللغة

Lack of native Arabic exception messages and interfaces. غياب الواجهات ورسائل الخطأ المترجمة للغة العربية.

Technical Architecture & Stack المعمارية البرمجية ومكونات التقنية

Technical Layer المكون البرمجي

The system utilizes React for a fast SPA client, Java Spring Boot for scalable enterprise business logic, and MySQL. يعتمد النظام على React لبناء واجهة سريعة أحادية الصفحة، و Spring Boot لإدارة منطق الأعمال وقواعد البيانات MySQL.

  • Frontend Layer: Vite, React, custom LTR/RTL CSS variables, and Leaflet Maps API. واجهة المستخدم: React مع Vite، وتنسيق ثنائي الاتجاه (LTR/RTL)، وخرائط Leaflet.
  • Backend Engine: Spring Boot REST APIs, JPA Hibernate ORM, and database scheduling logic. محرك الواجهة الخلفية: Spring Boot REST APIs، وجدولة قواعد البيانات عبر JPA.
  • Intelligent AI: Embedded Gemini API LLM concierge for conversational scheduling. الذكاء الاصطناعي: ربط مباشر بواجهة Gemini API للمساعد الذكي وجدولة الملاعب محادثياً.

Geofenced Attendance Verification التحقق الجغرافي من حضور الطلاب

Anti-Fraud Map منع التحايل الجغرافي

Ensures integrity by restricting check-ins to students physically inside the court geofence radius. يضمن مصداقية حضور الطلاب بقصر تسجيل الوصول على المتواجدين فعلياً ضمن النطاق الجغرافي للملعب.

  • 50-Meter Boundary: Check-in button locks unless GPS matches within 50m of facility center. نطاق الـ 50 متراً: زر تسجيل الحضور معطل تلقائياً إلا إذا تطابق موقع الهاتف مع حدود الملعب.
  • Auto No-Show Script: Runs 15 mins after booking starts, cancelling absent reservations. إلغاء حجز المتغيبين تلقائياً: يفحص النظام الحضور بعد 15 دقيقة، ويلغي حجز المتغيب مع إنذاره.
  • Suspension Rules: Accumulating 3 warnings results in a 1-week account suspension. عقوبات الغياب: عند تراكم 3 إنذارات عدم حضور، يتم تجميد حساب الطالب لمدة أسبوع.

LFG Matchmaking Queue طابور البحث عن لاعبين وتشكيل الفرق

Matchmaker تشكيل المجموعات

Bridges social gaps for solo players. Matches players anonymously, creates slots, and balanced rosters. يسهل الحجز للطلاب المنفردين دون حرج، بحشرهم في طابور بحث مجهول الهوية، وتوزيعهم لفرق متكافئة تلقائياً.

  • Privacy-First Queue: Student profiles masked until match confirmation to prevent social anxiety. طابور يحفظ الخصوصية: إخفاء هويات الطلاب حتى اكتمال الحجز لتشجيع الجميع على المشاركة.
  • Roster Generation: Splits matching queues into even Team A & Team B rosters. إنشاء قوائم الفرق: تقسيم اللاعبين بالتساوي لفرقتين متنافستين بشكل عادل.
  • Smart Match Prompt: Urges solo bookers to match rather than failing ID quota validation. التوجيه للطابور الذكي: يقترح النظام إحالة المستخدمين غير المكتملين لطابور البحث.

Seamless Gemini LLM integration allowing natural language bookings, rule queries, and account redirection. تكامل كامل مع نموذج Gemini للمحادثة، يتيح حجز الملاعب باللغة الدارجة، والاستفسار عن القوانين.

  • Gemini API integration: Processes complex queries and responds instantly in Arabic & English. ربط واجهة Gemini: معالجة الأسئلة المعقدة والاستجابة بطلاقة باللغتين العربية والإنجليزية.

  • Dynamic Theme Switch: Instant CSS layout adjustments using `.rtl-mode` styles and body `dir`. تغيير التنسيق الفوري: استجابة عناصر الواجهة لاتجاهات القراءة والخطوط المناسبة لكل لغة.
